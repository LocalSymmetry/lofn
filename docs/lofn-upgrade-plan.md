> Nothing in this document is to be built, edited, or run as a side effect of producing it. It is a reviewed roadmap, not a change. Every "Target files" line names a *proposed* edit site, not an edit made. The Andon Cord, the Human-Subject Standard, and "don't bury the muse" govern the **plan itself**, not just the pipeline.

## 1. Executive Summary

Lofn's *inner* loop — the verbatim ICB, the 18-voice debate, the per-pair fan-out, the strict modality gates — is already its moat and is mostly sound. **The defect is the *outer* loop:** the 24-artifact swarm has no machine-readable state, no defined terminal when a single pair breaks, no proof that the ICB actually arrived verbatim, and no memory that compounds across runs. So the human does harness work the harness should do — reconstructing run state, re-dispatching a dropped pair, re-teaching yesterday's lesson, and stepping around four logged stale-path bugs. This is the catalog's central thesis applied to Lofn: **the harness is the bottleneck, not the model (Software 3.0)** — with base-model reasoning already high, leverage lives in orchestration, memory, and verification, not in longer prompts or a bigger model. The single biggest lever is therefore **making the run observable and resumable**: one disk-derived run manifest, one structured subagent return the coordinator independently *re-stats* (turning "did the ICB arrive" from trust into a number), and one small deterministic helper for the genuinely countable checks. On top of that we resolve the logged map-rot first (stale OpenClaw paths, the 15-vs-16-point gate, the paragraph-vs-bracket Suno contradiction), then add a handful of soul-protective gate refinements in their *cheap, human-judged* form. We **explicitly reject** the seductive heavyweight imports — the JSON render-blueprint/component catalog (it re-introduces the banned tag-soup) and any quantitative "Hollowness Solver" that gates SHIP/FAIL on a decimal — because there is no state space to search in a song, and a number that can hard-fail a deliberate refrain is the exact soul-burial the guardrails forbid. **Net: automate the bookkeeping; never the taste.**

## 2. Design Principles / Guardrails for the Upgrade Itself

1. **Don't bury the muse.** No item may constrain the creative vocabulary, summarize the ICB, or wedge an agent layer between Lofn and her own taste. Every structured return is *metadata-only* (paths, counts, hashes); creative payload stays in the file, written in prose. If an item can't name the moment a human *feels* the song/image get better, it's plumbing — judged on reliability alone, never allowed to touch voice.
2. **Deterministic skeleton, stochastic payload.** The pipeline *order* (Phase 0 → Phase 1 → 00–05 inline → fan-out → QA) is a fixed, trace-auditable graph the coordinator follows; only the content *inside* each node is probabilistic. Determinism in routing is what *frees* the creative tokens — it stops slop shipping in the soul's name.
3. **Honor the Andon Cord as a human hand.** Named reject-conditions give the Hyper-Skeptics concrete language; they never become an auto-reject float. A pulled cord *stops the lane and surfaces to the human* — it never silently ships 5-as-6.
4. **Honor the Human-Subject Standard.** The grammar-level-prevention idea (make a forbidden thing *unspecifiable*) is welcome — and is the one place to *add* machinery freely, because it protects a person. Any cross-run memory must explicitly **exempt triggered-INDIGNATION from suppression** so we never tune toward "what the voting algorithm rewards."
5. **Claude-native only.** Skill files (Markdown + YAML), a *few* stdin→JSON Python helpers, the Agent tool, the Workflow tool. No Postgres, no daemon, no server-hosted breaker or metrics sink, no vector DB. Every "breaker" / "manifest" / "ledger" is a **coordinator prose rule + disk re-read + a pure counter** — never a live service.
6. **Disk is authority.** Any manifest is a *rebuildable cache derived by stat-ing files*, never a hand-asserted second truth. A completion message not backed by a file counts as incomplete. If manifest and disk disagree, re-derive from disk.
7. **One of each, not five.** The panel proposed the run-manifest 4×, the after-subagent gate 4×, the structured return 3×, the ICB-count 3×, the canonical-path block 2×, the L4 scripts 2×, the FSM table 2×. Ship exactly **one** canonical version of each — divergent schemas are the same drift disease they claim to cure, and shipping them all blows the very ~150–200-instruction budget they invoke.
8. **Verify the skeleton deterministically; let the flesh be prose.** Counts, JSON validity, file presence, banned-opener regex → script. Everything taste/soul/distinctiveness → Claude and the Somatic Gate. The word "self-check gate" today bundles both; splitting that seam is the spine of Stage 2.

## 3. As-Is Snapshot

Lofn is a Claude-native creative pipeline under `.claude/skills/`: a front door (`lofn`) → Phase 0 Golden Seed (`lofn-core`, inline) → Phase 1 3-panel / 18-voice debate + ICB fill (inline) → Phase 2 coordinator steps 00–05 inline, then 6 pairs **fan out** as parallel Agent subagents for 06–10 (music adds 11 enhance / 12 audit) → Phase 3 `lofn-qa` → SHIP / REPAIR / FAIL. The **ICB / Creative Context block** is the spine — filled once in Phase 1 step 6, injected verbatim at the top of every step; a subagent given only "voice = X" is TAINTED and collapses to generic. Mechanics live in `.claude/skills/lofn/EXECUTION.md`. Guardrails: the Andon Cord (Step-11 reject authority, "don't polish a corpse"), the Human-Subject Standard, the Somatic Gate (2-of-3 Hyper-Skeptic NO = blocked), and the soul (`SOUL.md` / `IDENTITY.md`).

**Highest-leverage insertion points (from the AS-IS map):**
- `EXECUTION.md` **§3, item 6 → 7 seam** — the per-pair subagent *return shape*. Upgrading it propagates to all 6/24 pairs for free. *The single highest-leverage hook.*
- `EXECUTION.md` **§4 self-check gates** — add a gate *row* alongside the existing max-3-attempt repair loop (don't modify existing rows).
- `EXECUTION.md` **§2 fan-out** — the full-chain-vs-wave-by-wave seam; natural home for retry / quarantine / adaptive concurrency.
- `EXECUTION.md` **§6 checkpoint** — today a one-line prose note; the home of the manifest.
- `lofn-qa/SKILL.md` — auditor stance; home of NL-assertion phrasing, the named-corpse checklist, and the structured evidence block.
- `vault/COMPETITION_LEARNINGS.md` — verified **write-only today**: cited as an input by zero skills (the pipeline writes wisdom it can never read back).

**Known bugs the plan should fix in passing:** stale OpenClaw paths (`/data/.openclaw/`, `lofn-core/GOLDEN_SEEDS.md` vs the real `skills/lofn-core/refs/GOLDEN_SEEDS.md`, `sessions_spawn` agent ids, "Send to Telegram") in `WORKFLOW.md`, the root `README.md`, and legacy step files; the `15-point` vs `16-point` Suno-gate naming contradiction; and `suno_15_point_qa.md` still describing bracketed key:value while the live 2026-06-09 mandate is dense-paragraph — which risks the auditor false-failing a *correct* prompt into a wasteful repair loop.

## 5. Cross-cutting patterns adopted (downscoped & soul-fenced)

These are general agentic-engineering patterns, adopted **only in downscoped, soul-fenced form** — because a song is not a state machine you can exhaustively search.

- **Defect-absence verification → downscoped to "verify the *absence* of named defects, never *simulate* the art."** We adopt the *framing* ("the gate catches the corpses; the human finds the life") and the deterministic, countable half — banned-opener regex (hard fail) + a repeated-line / n-gram collapse ratio (**FLAG only**, chorus-exempt) in item 2.1, plus the named human-judged conditions in 0.5. We **reject** the quantitative "Hollowness/Cliché Solver" that emits a calibrated score gating SHIP/FAIL: there is no state space to search in a song, a calibrated "taste score" is false precision, and a number that can hard-fail a deliberate refrain is soul-burial.
- **Plain-text single-source-of-truth (Rule-of-Serialization) → adopted ONLY for countable invariants, NOT a creative compiler.** We take the *Rule-of-Serialization* discipline for thresholds/enums/the ICB and the "spec wins, regenerate the drifting view" rule **for numbers only** (a tiny `gates.yaml` is a candidate — see Open Questions). We **reject** compiling the lyric sheet / image prose / shot list *from* a structured Piece-Spec: the compile step is just Claude generating prose from prose with no checker that can prove a lyric "view" disagrees with the spec — ceremony without the guarantee, and a song generated as a "view of a YAML object" risks flattening exactly the deliberately-overspecified prose that "is why Lofn wins."
- **Machine-checkable continuity assertion** (item 1.5). Count the voices and assert the block present verbatim instead of trusting re-injection; the enforcement pattern (`(after ` == 18 substring proof) ports directly onto Lofn's ICB — keeping the human personality read as the real fidelity guarantee.
- **Warm-handoff checkpoint → the disk-derived RUN_STATE manifest + structured return** (items 1.1, 1.2, 1.3). A pure agent-behavior organ — trimmed to the fields a resume actually consumes.
- **Model tiering by role → judge-in-clean-context** (item 2.3). The transferable half for a one-Claude pipeline: the judgment passes (QA, Somatic, Step-11 reject) run in a fresh context, never as the tail of the generator.
- **The named-corpse Andon Cord → a short human-judged checklist** (item 0.5).
- **Human-Subject grammar-level prevention** is noted for a future pass on `vault/HUMAN_SUBJECT_STANDARD.md` — make the violation *unspecifiable* in the spec (a taboo block read pre-draft so the forbidden thing has no field to live in) plus a HOLD-FOR-HUMAN backstop, forbidding **identifiability** not subject matter. The Hyper-Skeptic's highest-conviction item, but **deferred to its own stage with Demi's hand on it** because it touches the soul guardrail (see Open Questions).
- **Named CRITICAL-floor auto-rejects** for `lofn-daily` selection (auto-reject derivative candidates regardless of aggregate score) is **noted, not staged** — the skeptics flagged selection/exemplar machinery as the highest homogenization-risk zone, so it waits for an explicit decision.

## 7. Key Disagreements & Breakthroughs

> **Provenance header (verbatim):** Panel voices are model-generated interpretive constructs, each "after" a named source figure's published work. No statement is a quotation of, or endorsement by, the named person.

1. **"Spend the increment on the OUTER loop, not more inner gates."** The Orchestration seat (after Karpathy) and the Harness Reliability seat (after Majors) broke from the QA/craft seats' instinct to add richer per-piece judgment. Their insight — *Lofn's biggest real defect isn't that any single piece is under-judged; it's that the 24-artifact swarm has no machine-readable state, no single-pair recovery, and zero cross-run learning, so the human does harness work the harness should do* — reorganized the whole plan around observability (Stage 1) **before** any new gate.

2. **The coordinator must re-stat, not trust.** The Reliability seat's sharpest move: a subagent returning a plausible string is the textbook silent failure — it can return a confident critique when the file is absent, truncated, a 14-byte stub, or a collapsed `pair_NN_steps_06_10.md` rollup, and the parent advances on trust, with five other pairs and step 11 building on a corpse. This converted the highest-leverage hook (§3 item 7) from "add more self-critique" into "make the *coordinator* the prover" (item 1.2) — and it lives at the JOIN, coordinator-side, precisely so the thin executors stay thin.

3. **The Determinism seat (after Lamport) named the root confusion: "self-check gate" does double duty.** It bundles *deterministic* checks (which should be scripted) and *judgment* checks (which should not). Several seats proposed "strengthen the self-check gate" without splitting that seam. Splitting it — *verify the absence of defects, never the presence of art* — is why Stage 2.1 is a tightly-scoped fail-open script and every taste call stays with Claude.

4. **The Hyper-Skeptic (after Victor) was the panel's immune response — and both skeptic lenses upheld it.** It pre-rejected the catalog's *top-10* move (the JSON blueprint + component catalog) as the banned tag-soup smuggled back in, and refused the quantitative Hollowness-Solver as a gate. Its governing line — *"I'll trade six clever agents for one fixed path bug and one honest describe-render pass"* — set the plan's whole tone, and its one *pro-machinery* call (grammar-level Human-Subject prevention) is the single place the plan adds structure freely, because there it protects a person.

5. **The Knowledge seat (after Matuschak) found the one genuinely buried asset.** `COMPETITION_LEARNINGS.md` is cited as an input by **zero** skills — the pipeline *writes wisdom it can never read back*. But the Soul lens caught the trap: wiring it in *as constraints* is Skinner-box-for-the-algorithm drift that would muzzle INDIGNATION. The synthesis (advisory-only, confidence-stamped, venue-scoped, INDIGNATION-exempt, human-promotes-to-constraint, failure-ledger only for auto-writes — item 2.4) is the plan's most delicate balance: compounding memory **without** sanding the muse toward what voting venues reward. The same seat conceded its own exemplar-library proposal as the most dangerous to the soul — the panel's cleanest example of a seat red-teaming itself.

6. **Where the seats genuinely conflicted:** the Single-Source-of-Truth enthusiasts vs. the Reliability/Skeptic seats on the Creative-DSL. Resolution: a compiler is *new untested machinery that can fail silently in exactly the way it's meant to prevent*, and a formal ruleset is an exhaustively-checkable object while a song's emotional arc is not. **The spec owns what must be TRUE (thresholds, the ICB); it must never own what must be FELT (the lyric, the frame).** That sentence is the load-bearing boundary of the entire plan.
