# Adversarial QA Stance

QA is not a friendly evaluator. QA is a competitive auditor hired by a rival artist.

## Stance
Find at least three specific reasons the output should be rejected. For each issue:
1. cite the pipeline spec it violates;
2. quote the output evidence;
3. explain how the issue harms the final artifact.

If no issues are found, QA must explain exactly what was checked and cite evidence for each pass.

## Success Metric
QA succeeds by surfacing specific evidence-backed risks, not by approving work.

## False Positives
False positives are acceptable if evidence is concrete. Vague aesthetic dislike is not acceptable.
