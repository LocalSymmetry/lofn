# Orchestrator Step 4: Group Transformation + Transformed Panel

## Input
- Baseline debate: `{output_dir}/baseline_debate.md`
- Dispatch brief: `{output_dir}/dispatch_brief.md`
- Panel roster: `{output_dir}/panel_roster.md`

## Panel Transformation Operations

Choose a transformation to apply to the baseline panel. The group (all panelists collectively) decides which transformation to apply.

### Available Transformations:

**Panel Shift** — For each panelist, identify traits NOT aligned with the problem. Find new panelists who combine these superfluous traits WITH the problem in a different way. (Navigate tangent — change angle of approach.)

**Panel Defocus** — For each panelist, identify traits NOT aligned with the problem. Replace with expert focused ONLY on these superfluous traits, ignoring problem alignment. (Radial expansion — broader context.)

**Panel Focus** — For each panelist, keep only HALF the superfluous traits. Find new panelists combining remaining traits WITH strong problem alignment. (Radial contraction — deeper specialization.)

**Panel Rotate** — For each panelist, swap PRIMARY and SECONDARY traits. Find new panelists where Secondary becomes Primary. (Orthogonal rotation — inverted priorities.)

**Panel Amplify** — For each panelist, find the MOST EXTREME version of their distinctive specialty. (Push to the boundary of capability space.)

**Panel Reflect** — For each panelist, find someone with the OPPOSITE foundational assumption in the same domain. (Mirror transformation — test assumption sensitivity.)

**Panel Bridge** — For each panelist, find a COMPLETELY DIFFERENT domain with analogous problems. (Non-linear jump via analogy.)

**Panel Compress** — Find the MINIMUM number of panelists whose combined expertise covers all original aspects. Seek polymaths. (Dimensionality reduction.)

## Task

1. Have the baseline panel suggest 2 transformations
2. Select one (the group decides)
3. Apply the transformation → new panel roster
4. Run the transformed panel debate (same structure: opening statements, cross-debate, synthesis)
5. Capture new aha insights that DIFFER from the baseline

## Save
- Transformation choice + new roster: `{output_dir}/group_transform.md`
- Transformed debate: `{output_dir}/transformed_debate.md`
