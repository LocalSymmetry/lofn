# Release / Falsification Logging

Every published or candidate song/image should log the hypothesis before release. This prevents post-hoc story-making.

## Required Questions
- What route was intended: ACCESSIBLE or AMBITIOUS?
- Which eligibility properties were targeted and scored pre-release?
- Which archetype and seed source were used?
- What AI/technical vocabulary disclosure level was used?
- What distribution event occurred, if any?
- What metrics were observed at 7/14/30 days?

## Interpretation
- Eligibility score estimates P(eligible), not guaranteed success.
- Staff Pick / playlist / influencer event is distribution, not intrinsic song quality.
- Amplification is measured via retention, likes, saves/shares, comments, playlist adds.

## Falsification Discipline
A failed high-eligibility release should not be explained away automatically. Track the failure and compare across at least 10 releases before revising the formula.
